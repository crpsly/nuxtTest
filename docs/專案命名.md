
# Nuxt 3 專案命名慣例

**核心原則：**

*   **一致性 (Consistency)：** 在整個專案中堅持使用同一種風格。
*   **可讀性 (Readability)：** 命名應清晰地表達檔案的用途。
*   **普遍性 (Convention)：** 遵循 Vue/Nuxt 和 JavaScript 社群的普遍慣例。

**各目錄命名建議 (含兩個單詞示例)：**

1.  **`components/` (元件)**
    *   **命名法：** `PascalCase` (大駝峰式命名法)
    *   **說明：** 這是 Vue 元件的標準且強烈建議的命名方式。它清楚地區分元件檔案，並與元件在模板中的使用方式 ( `<UserProfile />` ) 保持一致。
    *   **示例：**
        *   `UserProfile.vue`
        *   `BaseButton.vue`
        *   `ModalDialog.vue`
        *   `ProductCard.vue`
    *  **補充說明：**
        - nuxt.config.ts: components: [{ pathPrefix: false }], 這樣若是多層級資料夾，則不會將資料夾名稱也一併當作元件名稱的一部分。
        - 例如：`components/user/Profile.vue`，則會被當作 `Profile.vue`，而不是 `UserProfile.vue`。

2.  **`pages/` (頁面)**
    *   **命名法：** `kebab-case` (短橫線分隔命名法)
    *   **說明：** Nuxt 的檔案系統路由會將此命名轉換為 URL 路徑 (例如 `user-profile.vue` -> `/user-profile`)。使用小寫和短橫線是 URL 的標準格式。
    *   **示例：**
        *   `user-profile.vue`
        *   `contact-us.vue`
        *   `product-list.vue`
        *   `about-us.vue`
        *   `terms-conditions.vue`
    *  **補充說明：**
        - 資料夾路徑的單詞若要分隔也使用 - 短橫線，例如：`pages/user-profile/[id].vue`，則會被當作 `/user-profile/`，而不是 `/userProfile/`。

3.  **`layouts/` (佈局)**
    *   **命名法：** `kebab-case` (短橫線分隔命名法)
    *   **說明：** 雖然 `default.vue` 是預設佈局的特殊名稱，但自訂佈局建議使用 `kebab-case`，與 `pages/` 保持一致性。
    *   **示例：**
        *   `admin-layout.vue`
        *   `guest-layout.vue`
        *   `mobile-view.vue`
    *  **補充說明：** 在子資料夾中的 layout 命名，參考以下範例：

        | 文件                        | 佈局名稱       |
        | :------------------------- | :------------ |
        | ~/layouts/desktop/DesktopDefault.vue | desktop-default |
        | ~/layouts/desktop-base/DesktopBase.vue | desktop-base |
        | ~/layouts/desktop/Desktop.vue | desktop |

4.  **`composables/` (組合式函數)**
    *   **命名法 (檔案名)：** `kebab-case` (短橫線分隔命名法) *或* `camelCase` (小駝峰式命名法)。
    *   **示例 (檔案名)：**
        *   `use-user-auth.ts`
        *   `use-fetch-data.ts`
        *   `useWindowSize.ts`
        *   `useFormValidation.ts`
    *   **示例 (對應導出函數名)：**
        *   `useUserAuth`
        *   `useFetchData`
        *   `useWindowSize`
        *   `useFormValidation`

5.  **`middleware/` (中介軟體)**
    *   **命名法：** `kebab-case` (短橫線分隔命名法)
    *   **說明：** 中介軟體通常按名稱引用，`kebab-case` 清晰易讀。Nuxt 3 還支持 `.global` 後綴表示全域中介軟體。
    *   **示例：**
        *   `auth-guard.ts`
        *   `guest-only.ts`
        *   `request-logger.global.ts`
        *   `permission-check.ts`
    *  **補充說明：** 特定的順序會影響中介軟體的執行順序，這在 Nuxt 3 中是由檔案名稱的字母順序決定的。建議在全域中間件前加上「字母」編號。
        例如：`01-auth-guard.ts`，這樣可以確保它在其他中介軟體之前執行。

6.  **`plugins/` (插件)**
    *   **命名法 (檔案名)：** `kebab-case` (短橫線分隔命名法) *或* `camelCase` (小駝峰式命名法)。
    *   **說明：** 你可以在檔案名稱中使用 .server 或 .client 後綴，來讓插件僅在伺服器端或客戶端載入。
    *   **說明：** 所有放在 plugins 資料夾內的插件都會自動註冊，無需另外在 nuxt.config 中手動加入。

7.  **`server/api/` (伺服器 API 路由)**
    *   **命名法：** `kebab-case` (短橫線分隔命名法)
    *   **說明：** 這直接映射到 API 的 URL 路徑。Nuxt 3 會自動處理 `.get`, `.post` 等 HTTP 方法後綴。
    *   **示例：**
        *   `user-data.get.ts`
        *   `product-list.get.ts`
        *   `submit-form.post.ts`
        *   `session-info.get.ts`

8.  **`server/middleware/` (伺服器中介軟體)**
    *   **命名法：** `kebab-case` (短橫線分隔命名法)
    *   **說明：** 與客戶端中介軟體類似，用於處理伺服器請求。
    *   **示例：**
        *   `log-requests.ts`
        *   `cors-handler.ts`
        *   `auth-verify.ts`

9.  **`server/utils/` 或 `utils/` (輔助工具函數)**
    *   **命名法：** `kebab-case` (短橫線分隔命名法)
    *   **說明：** 用於存放通用的輔助函數模組。
    *   **示例：**
        *   `date-format.ts`
        *   `string-helpers.ts`
        *   `validation-rules.ts`
        *   `api-helpers.ts`

10. **`assets/` (靜態資源 - 需要建構)**
    *   **命名法：** `kebab-case` (短橫線分隔命名法)
    *   **說明：** 用於存放需要 Webpack 或 Vite 處理的資源，如 SASS/LESS 檔案、需要優化的圖片等。
    *   **示例：**
        *   `main-styles.scss`
        *   `variable-definitions.scss`
        *   `logo-dark.png`
        *   `hero-image.jpg`

11. **`public/` (公共資源 - 直接複製)**
    *   **命名法：** `kebab-case` (短橫線分隔命名法)
    *   **說明：** 這裡的檔案會被直接複製到輸出目錄的根目錄下，通常用於 `robots.txt`, `favicon.ico`, 不需處理的圖片等。
    *   **示例：**
        *   `site-icon.png`
        *   `service-worker.js` (如果手動管理)
        *   `ads-txt.txt` (雖然通常是 `ads.txt`)

**總結表格：**

| 目錄                 | 命名法        | 說明                               | 兩個單詞示例                     |
| :------------------- | :------------ | :--------------------------------- | :------------------------------- |
| `components/`        | `PascalCase`  | Vue 元件標準                       | `UserProfile.vue`, `BaseButton.vue` |
| `pages/`             | `kebab-case`  | 對應 URL 路徑                      | `user-profile.vue`, `contact-us.vue` |
| `layouts/`           | `kebab-case`  | 自訂佈局，與 pages 一致            | `admin-layout.vue`, `guest-layout.vue` |
| `composables/` (檔案) | `kebab-case`  | 輔助函數檔案，一致性             | `use-user-auth.ts`, `use-fetch-data.ts` |
| `middleware/`        | `kebab-case`  | 中介軟體，按名稱引用               | `auth-guard.ts`, `guest-only.ts`    |
| `plugins/`           | `kebab-case` 或 `camelCase`  | 插件檔案           | `axios-setup.ts`, `errorHandler.client.ts` |
| `server/api/`        | `kebab-case`  | 對應 API URL 路徑                  | `user-data.get.ts`, `product-list.get.ts` |
| `server/middleware/` | `kebab-case`  | 伺服器端中介軟體                   | `log-requests.ts`, `cors-handler.ts` |
| `utils/`             | `kebab-case`  | 通用輔助函數模組                   | `date-format.ts`, `string-helpers.ts` |
| `assets/`            | `kebab-case`  | 需要建構處理的靜態資源             | `main-styles.scss`, `logo-dark.png` |
| `public/`            | `kebab-case`  | 不需處理，直接複製的公共資源       | `site-icon.png`, `service-worker.js` |

遵循這些建議可以讓你的 Nuxt 3 專案更加結構化、易於維護和理解。
